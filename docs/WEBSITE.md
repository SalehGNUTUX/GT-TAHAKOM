# موقع المشروع — البناء والتحديث

الموقع مستضاف على **GitHub Pages**: https://salehgnutux.github.io/GT-TAHAKOM/
يُقدَّم من جذر فرع `main` (إعداد Pages: branch=main, path=/). ملف `.nojekyll` يمنع
معالجة Jekyll فيُقدَّم HTML كما هو.

## البنية
- `index.html` (الجذر): صفحة الهبوط **المستقلّة بذاتها** (CSS/JS مضمّن، بلا تبعيات خارجية
  عدا خط Google) — شعار بأسلوب بطاقة هوية الإعدادات (أيقونة + اسم + عبارة، تأثير عند المرور
  ونقرة تُنعِش الصفحة) + بطل + مزايا + **نموذج حيّ يحاكي شاشة الريموت الفعلية** (HTML/CSS
  أصلي، أيقونات Material عبر SVG، مع تأثيرات ماتيريال: تموّج، ارتفاع، ظهور تدريجي). ثنائية
  اللغة (ع/EN) ووضعان (فاتح/داكن) بنفس رموز التطبيق (Tokens)، تُحفظ التفضيلات في localStorage.
- `website/assets/`: أيقونة التطبيق.
- (`website/demo/` القديم — نموذج React من مجلّد التصميم — لم يَعُد مستخدَماً؛ النموذج الآن مضمّن
  في `index.html` ويحاكي التطبيق الفعلي مباشرةً.)

## التحديث مع كل إصدار
- **شارة الإصدار تتحدّث تلقائياً** من آخر إصدار منشور عبر GitHub API (مع بقاء `v1.0.0` كقيمة احتياطية).
- **زر التحميل** يشير دائماً لأحدث إصدار (`releases/latest`).
- لميزة جديدة: أضِفها إلى `I18N.ar.feats` و`I18N.en.feats`.
- لتغيير في شاشة الريموت بالتطبيق: حدّث قسم `.phone` في `index.html` (الأزرار/الأيقونات) ليطابقه.
- ادفع إلى `main` — تتحدّث Pages تلقائياً خلال دقيقة.

## التحقّق
```bash
curl -s -o /dev/null -w "%{http_code}" https://salehgnutux.github.io/GT-TAHAKOM/
gh api repos/SalehGNUTUX/GT-TAHAKOM/pages/builds/latest --jq .status   # = built
```
