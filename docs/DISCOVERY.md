# الاكتشاف الذكي للأجهزة — كيف يحدّد GT-TAHAKOM الجهاز تلقائياً؟

سؤال جوهري: **هل يمكن إرسال إشارة، ومن ردّ الجهاز نحدّد نوعه ونضبطه دون تدخّل المستخدم؟**
الجواب يعتمد على وسيلة الاتصال، وهذا أساس تصميم التطبيق.

## 1. الأجهزة الشبكية (WiFi/LAN) — اكتشاف تلقائي كامل ✅

تشمل: التلفازات الذكية، **صناديق أندرويد (الأصلية والصينية)**، أجهزة الاستقبال الرقمية بالشبكة،
Roku، Chromecast. كل هذه **تعلن عن نفسها** على الشبكة، فنكتشفها ونحدّد بروتوكولها آلياً:

| الآلية | ماذا تكشف | يطبّقها |
| :-- | :-- | :-- |
| **mDNS / NSD** (Bonjour) | أنواع خدمات مثل `_androidtvremote2._tcp`، `_googlecast._tcp`، `_roku._tcp` + سجلات TXT (الطراز/العلامة) | `MdnsDiscovery` (م1) |
| **SSDP / UPnP** | ترويسات `SERVER` و`manufacturer` و`modelName` (Samsung/LG/Sony يكشفون أنفسهم) | `SsdpDiscovery` (م1) |

ثم يحوّل [`ServiceFingerprint`](../app/src/main/java/com/gnutux/tahakom/core/discovery/ServiceFingerprint.kt)
هذا الإعلان إلى استنتاج: **أي `TransportType`؟ أي `DeviceType`؟ أي علامة؟** — فيُضاف الجهاز ويُضبط مباشرة.

> **الصناديق الصينية:** تعمل بنظام أندرويد (AOSP) فتُعلن مثل الأصلية عبر `_androidtvremote2` أو
> `_googlecast`؛ تُكتشف بنفس الطريقة. ما لا يُعلن منها يُضاف يدوياً بالـ IP.

## 2. أجهزة الأشعة تحت الحمراء (IR) — لا اكتشاف تلقائي ❌ (قيد فيزيائي)

التلفاز القديم/الريسيفر بريموت أحمر يُستقبَل بـ IR، و**الـ IR أحادي الاتجاه**: التطبيق يرسل فقط
ولا يستقبل أي ردّ. لا توجد قناة رجوع، فمن المستحيل تقنياً «إرسال إشارة ومعرفة من استجاب».
هذا قيد على كل تطبيقات الريموت بلا استثناء.

**الحل المعتمد عالمياً (سنطبّقه في م4): الاكتشاف شبه الآلي بالتأكيد البصري**
1. يرسل التطبيق أمر `POWER` لكل علامة بالتتابع (Samsung → LG → Sony → ...).
2. يسأل المستخدم: «هل استجاب الجهاز؟».
3. عند «نعم» تُحدَّد العلامة (وغالباً مجموعة الأكواد/الطراز) ويُضبط الريموت.

هذه أسرع طريقة ممكنة فيزيائياً لـ IR، وهي تجربة «شبه تلقائية» بضغطة تأكيد واحدة.

## 3. جسر WiFi-IR (Broadlink) — هجين

جهاز Broadlink متصل بالـ WiFi (فيُكتشف شبكياً) لكنه يبثّ IR للجهاز الهدف. لذا:
- اكتشاف الجسر نفسه: **تلقائي** (شبكي).
- تحديد الجهاز الهدف خلف الجسر: **شبه آلي** (نفس منطق IR أعلاه)، لكنه يعمل من **أي هاتف** بلا باعث.

## الخلاصة

| نوع الجهاز | اكتشاف تلقائي بلا تدخّل؟ |
| :-- | :-- |
| تلفاز ذكي / Android Box (أصلي/صيني) / ريسيفر IP | ✅ نعم (mDNS/SSDP) |
| تلفاز/ريسيفر IR تقليدي | ❌ لا (قيد فيزيائي) → شبه آلي بضغطة تأكيد |
| عبر جسر Broadlink | الجسر تلقائي، الهدف شبه آلي |
