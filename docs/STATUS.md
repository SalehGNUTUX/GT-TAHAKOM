## نظرة عامة

GT-TAHAKOM تطبيق أندرويد (Kotlin + Jetpack Compose) للتحكّم في التلفاز والأجهزة
الإلكترونية عبر وسائل متعددة، يعمل **أوفلاين بالكامل** للوظائف الأساسية. عربي/إنجليزي
مع RTL. الجوهر المعماري: **طبقة `Transport` مجرّدة** تختار الوسيلة المناسبة لكل جهاز.

## ما أُنجز ✅

### البنية والتأسيس (م0)
- مشروع Kotlin + Compose + Material 3، Hilt، Room/DataStore، OkHttp/Coroutines.
- طبقة `Transport` المجرّدة + `TransportRegistry` + `TransportResult`.
- النموذج الموحّد: `Device` / `Command` / `ButtonId` / `Remote`.
- توقيع release + بناء APK (`docs/BUILD.md`).

### الاكتشاف التلقائي للأجهزة الشبكية (م1) — أوفلاين
- `MdnsDiscovery` (NsdManager): Android TV/Box، Cast، Roku، AirPlay.
- `SsdpDiscovery` (UDP multicast): Samsung/LG/Sony عبر UPnP.
- `DiscoveryManager` (دمج + MulticastLock) + `ServiceFingerprint` (استنتاج البروتوكول/العلامة).
- شاشة أجهزة حيّة. (التفاصيل: `docs/DISCOVERY.md`).

### التحكّم الفعلي عبر الشبكة (م2 + م3 جزئياً)
- `RokuTransport` (ECP/HTTP) — إرسال شبكي حقيقي بلا إقران.
- `WebosTransport` (LG webOS / SSAP WebSocket على :3000) — **يحلّ ما يعجز عنه IR**:
  فتح التطبيقات/الإعدادات الذكية، الوسائط، الصوت/القناة، عبر أوامر `ssap://` صريحة.
  إقران بـ client-key محفوظ (`WebosKeyStore`). التنقّل (الاتجاهات/OK/رجوع) عبر pointer
  input socket — مكتمل.
  ⚠️ تجربة ميدانية (LG @192.168.1.17) كشفت علّتين تُعالَجان في الإصدار القادم — راجع
  [TEST_NOTES.md](TEST_NOTES.md#2): إطلاق التطبيقات (payload لا query)، والتنقّل/لوحة اللمس (اتصال دائم).
- `SamsungTizenTransport` (Samsung Tizen / WebSocket wss على :8002) — مفاتيح KEY_*
  (طاقة/صوت/قناة/تنقّل/تطبيقات/ملوّنة)، token محفوظ بعد قبول المستخدم، TLS متساهل محلياً.
- إعدادات كاملة: سمة فاتح/داكن/نظام (محفوظة DataStore) + اللغة + معلومات الوسائل + حول.
- مصطلحات: «جهاز تحكّم» بدل «ريموت» في كل نصوص الواجهة.
- `RemoteScreen` — شاشة ريموت عامة، تعرض لكل بروتوكول أزراره المدعومة.
- إعداد أمان الشبكة (`network_security_config`) للسماح بـ ws:// المحلي.

### طبقة IR + القاعدة المحلية (م4 جزئياً) — أوفلاين
- قاعدة `assets/irdb/`: **67 جهازاً** (TV 35 + Cable 25 + Audio 7) من IRRemote + استيراد
  NEC/RC5/RC6 من probonopd/irdb، بصيغة موحّدة (index.json + ملف/جهاز، أكواد Pronto + تردد).
  `tools/convert_ir_db.py` + `tools/import_irdb.py` (يطبّع أسماء أزرار LIRC ‏KEY_*‎ و‏CURSOR‎).
- **ضبط IR بتنقّل تالي/السابق**: استعراض مرشّحي الفئة جهازاً تلو الآخر دون مغادرة شاشة الضبط.
- `IrDatabase` (قارئ + بحث)، `Pronto` (محوّل Pronto→تردد+نمط)، `IrCommandResolver`.
- `IrTransport` — إرسال فعلي عبر `ConsumerIrManager`.
- **ضبط IR شبه آلي** بمنطق حالة الطاقة (مطفأ→Power، مشغّل→Vol/Ch). (`docs/DISCOVERY.md`).

### المزايا
- **التعلّم بالإدخال اليدوي** — دعم الأجهزة غير الموثّقة (Unionaire…) بإدخال أكواد
  Pronto، تُحفظ كأجهزة IR مخصّصة وتُدمج في القاعدة. (`docs/LEARN.md`).
- **البحث الشبكي عن أجهزة التحكّم** (الطبقة «التوسعية») — هجين أوفلاين-أولاً: فهرس
  probonopd كامل (3244 طقماً، **2584 مدعوماً**) مشحون كأصل فالبحث بالعلامة أوفلاين (مع مرشّح
  نوع الجهاز: الكل/تلفاز/استقبال/صوت)؛ تنزيل
  أكواد الطقم المختار عبر CDN ثم **تحويله إلى Pronto على الهاتف** (`IrCodeConverter`،
  NEC/RC5/RC6/Sony SIRC/Panasonic/JVC/Mitsubishi/Denon/**Pioneer**/**Proton**، **2584/3244 مدعوم**)
  وحفظه كجهاز محلي. توليد الفهرس بـ `tools/build_online_index.py`.
- **لوحة اللمس** في شاشة الريموت للأجهزة الشبكية: سحب للتنقّل + نقر للاختيار (NAV_* عبر pointer socket).
- **تبديل اللغة** عربي/إنجليزي من الإعدادات (per-app locale).
- **مشاركة حِزم `.tahakom`** (علامة/طراز) + الفتح المباشر بالنقر. (`docs/SHARING.md`).
- **قائمة "أجهزتي"** المحفوظة (DataStore): فتح/مشاركة/حذف.
- **شاشة ترحيب** (3 شرائح) عند أول تشغيل.

### التصميم (جزئياً) — اعتماد GT-TAHAKOM-DESIGN
- نظام رموز serene (ألوان OKLCH→sRGB، فاتح/داكن، ألوان الوسائل) — `ui/theme/Tokens.kt`.

- **مدفوع بالبيانات**: كل جهاز يعرض ما يدعمه فقط. (`docs/REMOTE_BUTTONS.md`).

## ملاحظات تقنية حاسمة (دروس)
- **AppCompatActivity تتطلّب ثيم `Theme.AppCompat`** وإلا انهيار فوري (تبديل اللغة يحتاجها).
- **لا تكتب محلّل مسارات SVG يدوياً** لـ ImageVector — أعلام القوس الملتصقة تنهار وقت التشغيل. استخدم Material Icons.
- المنصة المحلية: android-36 (لا 35) → compile/target=36، AGP 8.9.2.
- اكتشاف IR التلقائي مستحيل فيزيائياً (أحادي الاتجاه) → شبه آلي بالتأكيد البصري.
- **أوفلاين افتراضياً**: لا وظيفة أساسية تعتمد على الإنترنت. (`docs/DATABASE.md`).
